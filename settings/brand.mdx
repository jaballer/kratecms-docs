KrateCMS gives you full white-label control over how your site looks. From the Brand settings page you can change your color scheme, upload your logo and favicon, select your fonts, toggle dark mode, and even inject your own CSS rules. Changes apply instantly — no deploy or rebuild required. Color, font, and theme changes apply to both your public site and your admin panel; Custom CSS applies to your public site only.

## Getting there

Navigate to **Settings → Brand** in your admin sidebar, or go directly to `/admin/settings/brand`.

## Colors

The **Primary color**, **Secondary color**, and **Accent color** fields accept any valid hex color value (for example, `#4c44e4`). The primary color is used for the most prominent interactive elements — main buttons, active navigation links, and focus states. The secondary color provides a complementary tone for surfaces and supporting UI elements. The accent color is used for highlights and decorative elements throughout the UI.

<Steps>
  <Step title="Open Brand settings">
    Go to **Settings → Brand**.
  </Step>
  <Step title="Enter your colors">
    Click the color swatch next to **Primary color**, **Secondary color**, or **Accent color** to open the color picker, or type a hex value directly into the input field.
  </Step>
  <Step title="Save">
    Click **Save**. Your new colors apply immediately across your public site and the admin.
  </Step>
</Steps>

## Typography

Select a **Heading font** and a **Body font** independently. Your heading font applies to all `h1`–`h6` elements across the site, while your body font applies to paragraphs, navigation labels, and general UI text.

<Note>
Font choices are applied globally. If you need per-block or per-page typography overrides, use the Custom CSS field described below.
</Note>

## Theme mode

Use the **Theme mode** toggle to switch your site between **Light** and **Dark** mode. The selected mode applies to both the public-facing site and the admin interface.

<Tabs>
  <Tab title="Light mode">
    The default. White and light-grey backgrounds with dark text. Suitable for most general-purpose sites.
  </Tab>
  <Tab title="Dark mode">
    Dark backgrounds with light text. Well-suited for media-heavy sites, tech publications, or audiences who prefer lower-contrast interfaces.
  </Tab>
</Tabs>

## Logo

Upload your site logo to have it displayed in the navigation bar and header on every page. KrateCMS accepts common image formats including PNG, SVG, JPG, and WebP.

When you choose a file, a live preview appears so you can confirm it before saving. Once uploaded, your logo is stored in cloud storage and served from there. You do not need to re-upload it if you change other settings.

<Note>
For the sharpest results on all screen sizes and resolutions, use an SVG or a PNG with a transparent background. Aim for a height of around 40–60 px at 2× resolution.
</Note>

## Favicon

Your favicon is the small icon that appears in the browser tab and in bookmarks. Upload a square image — ideally a 32×32 or 64×64 PNG, or an `.ico` file. It previews live when you select a file. Like the logo, your favicon is served from cloud storage after upload.

## Custom CSS

The Custom CSS field lets you write arbitrary CSS rules that are injected into every **public** page of your site. This is the escape hatch for any visual customization that the standard settings do not cover — adjusting spacing, overriding component styles, or adding brand-specific flourishes.

To target a specific page or an individual block, use the per-page and per-block **CSS hooks** — automatic selectors like `#page-{id}` and `.page-block--{type}`, plus the **HTML ID** / **CSS Classes** fields on each block. See [Style blocks with CSS hooks](/content/pages#style-blocks-with-css-hooks) for the full list.

<Tip>
Custom CSS is powerful, but it runs on every public page of your site. (It is no longer injected into the admin panel, so a styling mistake can't lock you out of your own editor.) Test your rules carefully with a narrow selector before saving — broad selectors like `* { ... }` can have unintended side effects.
</Tip>

```css
/* Example: increase the default body font size */
body {
  font-size: 17px;
}

/* Example: add a custom border to post cards */
.post-card {
  border: 2px solid var(--color-primary);
  border-radius: 8px;
}
```

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Do brand changes affect the admin as well as the public site?">
    Color, typography, and theme-mode changes apply to both the public-facing site and the admin panel simultaneously. Custom CSS is the exception — it is applied to your public pages only, never the admin, so a bad rule can't break your editor.
  </Accordion>
  <Accordion title="Can I revert to the default logo or favicon?">
    Yes. Open Brand settings and use the remove or replace control next to the logo or favicon upload. Saving with no image uploaded returns to the default placeholder.
  </Accordion>
  <Accordion title="How do I use a different color for a specific section?">
    Use the Custom CSS field to target the section with a CSS selector and apply your color there. For example: `.hero-section { background-color: #f0f0f0; }`.
  </Accordion>
  <Accordion title="My Custom CSS is breaking the layout — how do I fix it?">
    Open Brand settings, clear the Custom CSS field, and click **Save**. This removes all custom CSS immediately. Then re-add your rules incrementally to isolate the problematic rule.
  </Accordion>
</AccordionGroup>
